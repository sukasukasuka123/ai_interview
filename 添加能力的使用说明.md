# 添加新能力的使用说明

## 架构总览

```
main.py
  ├── HelperEngine   → Agent(ASSISTANT_SKILLS)  → AI 助手
  └── InterviewEngine → Agent(INTERVIEW_SKILLS) → 面试引擎
```

```
UI 层               引擎层                   工具层
────────────        ─────────────────        ──────────────────────────────────────
AgentPanel        → HelperEngine           → ASSISTANT_SKILLS
                      Agent                    COMMON_GROUP  → get_job_position_info
                                                              → draw_questions_from_bank
                                                              → get_question_bank_stats
                                               QUIZ_GROUP    → search_question_bank
                                               RAG_GROUP     → search_knowledge_base → KnowledgeCore(TECH_KB_ID)
                                               SEARCH_GROUP  → web_search, search_wikipedia
                                               HISTORY_GROUP → get_student_interview_history
                                                              → get_student_id_by_name

InterviewPanel    → InterviewEngine        → INTERVIEW_SKILLS
                      Agent                    COMMON_GROUP  → (同上)
                                               DS_COURSE_GROUP → search_ds_course → KnowledgeCore(DS_COURSE_KB_ID)
```

### 各层职责

**`main.py`** — 组装层，实例化引擎和 UI，只做连线，无业务逻辑

**引擎层**（`helper_engine.py` / `interview_engine.py`）— Agent 的使用方，管理业务状态（面试 session、对话历史）和 system prompt

**`agent_core.py`** — 纯 LLM + tool_calling 框架，不含业务逻辑，对外暴露 `stream()` / `chat()` / `clear_conversation()`

**工具层** (`service/tools/`):
- `permissions.py` — ToolGroup 路由组 + SkillSet 组合，只声明"谁有哪些工具"
- `registry.py` — 工具工厂组装中心，按 SkillSet 筛选返回工具列表
- `db_tools.py` — 数据库类工具工厂函数
- `search_tools.py` — 联网搜索类工具工厂函数
- `knowledge/` — 知识库能力子模块
  - `KnowledgeCore.py` — 百炼 RAG SDK 封装，认证信息从 env 读取，`knowledge_base_id` 通过构造函数注入
  - `create_knowledge_search_tool.py` — `search_knowledge_base`（HelperEngine 用）
  - `create_ds_course_tool.py` — `search_ds_course`（InterviewEngine 用）

### KnowledgeCore 与工具的关系

每个知识库工具文件持有一个 `KnowledgeCore` 实例。工厂函数签名为 `factory(kb=None)`：
- 调用时传入 `kb` 实例 → 使用传入的实例
- 传入 `None`（默认）→ 工厂内部从对应 env 变量自动构造

所以引擎层和 `main.py` **不需要**手动创建 `KnowledgeCore`，直接调用 `get_interview_tools(db)` 或 `get_assistant_tools(db)` 即可。

## 添加新能力的步骤

### 情况一：新增一个普通工具（DB 查询、搜索等）

**第一步** — 在 `db_tools.py` 或 `search_tools.py` 写工厂函数

```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class YourToolInput(BaseModel):
    param: str = Field(..., description="参数描述，模型靠这段话决定传什么值")

def create_your_tool(db):
    @tool(args_schema=YourToolInput)
    def your_tool_name(param: str) -> str:
        """工具描述，模型靠这段话决定是否调用"""
        # 实现逻辑
        return result
    return your_tool_name
```

**第二步** — 在 `permissions.py` 加常量并放入对应 ToolGroup

```python
# 加工具名常量
TOOL_YOUR = "your_tool_name"

# 加进合适的 ToolGroup
COMMON_GROUP = ToolGroup(
    name="common",
    tools=frozenset({TOOL_JOB_INFO, TOOL_QUIZ_DRAW, TOOL_QUIZ_STATS, TOOL_YOUR}),
)
```

**第三步** — 在 `registry.py` 的 `build_tools()` 注册工厂

```python
_db_factories = [
    # ...原有条目...
    ("your_tool_name", create_your_tool),   # ← 加这里
]
```

SkillSet 自动更新，无需改引擎或 UI。

### 情况二：新增一个知识库（新的 RAG 检索能力）

**第一步** — 在 `knowledge/` 下新建工具文件

```python
# knowledge/create_xxx_kb_tool.py
import os
from langchain_core.tools import tool
from pydantic import BaseModel, Field
from service.tools.knowledge.KnowledgeCore import KnowledgeCore

class XxxKbInput(BaseModel):
    query: str = Field(..., description="检索关键词")
    top_k: int = Field(default=3, ge=1, le=5)

def create_xxx_kb_tool(kb: KnowledgeCore = None):
    if kb is None:
        kb_id = os.getenv("XXX_KB_ID", "")
        if not kb_id:
            raise ValueError("create_xxx_kb_tool：XXX_KB_ID 未配置")
        kb = KnowledgeCore(knowledge_base_id=kb_id, label="XXX知识库")

    @tool(args_schema=XxxKbInput)
    def search_xxx(query: str, top_k: int = 3) -> str:
        """描述这个知识库的内容和适用场景，模型靠这段话决定是否调用"""
        results = kb.retrieve(query, top_k=top_k)
        if not results or results[0].startswith(("📭", "⚠️")):
            return results[0] if results else "未找到相关内容。"
        lines = [f"检索结果（{query}）：\n"]
        for i, r in enumerate(results, 1):
            lines.append(f"[{i}] {r}\n")
        return "\n".join(lines)

    return search_xxx
```

**第二步** — 在 `knowledge/__init__.py` 导出

```python
from .create_xxx_kb_tool import create_xxx_kb_tool
```

**第三步** — 在 `permissions.py` 加常量和 ToolGroup

```python
TOOL_RAG_XXX = "search_xxx"

XXX_KB_GROUP = ToolGroup(
    name="xxx_kb",
    tools=frozenset({TOOL_RAG_XXX}),
)

# 把新 Group 加进需要的 SkillSet
ASSISTANT_SKILLS = SkillSet.from_groups(
    "assistant", "...",
    COMMON_GROUP, QUIZ_GROUP, RAG_GROUP, XXX_KB_GROUP,  # ← 加这里
    SEARCH_GROUP, HISTORY_GROUP,
)
```

**第四步** — 在 `registry.py` 的 `_kb_factories` 加一行

```python
_kb_factories = [
    ("search_knowledge_base", create_knowledge_search_tool, tech_kb),
    ("search_ds_course",      create_ds_course_tool,        ds_course_kb),
    ("search_xxx",            create_xxx_kb_tool,           None),   # ← 加这里，None 表示自动读 env
]
```

**第五步** — 在 `.env.example` 加新变量

```env
XXX_KB_ID="xxxxxxxxxxxxxxxxx"
```

引擎层和 `main.py` 不需要任何改动。

---

### 情况三：新增一个引擎（新的业务场景）

1. 新建 `xxx_engine.py`，持有 `Agent` 实例，在 `__init__` 中调用 `get_tools_for(db=db, skill_set=XXX_SKILLS)` 加载工具
2. 在 `permissions.py` 按需新增 SkillSet 组合
3. 在 `UI/` 新建对应 Panel，接入引擎流式接口
4. 在 `main.py` 实例化引擎和 Panel，挂载到 TabWidget

---

## 一句话总结

> **加工具** → `db_tools.py`/`search_tools.py` 写工厂 + `permissions.py` 加常量 + `registry.py` 注册
> **加知识库** → `knowledge/` 新建文件（内部自动读 env）+ `permissions.py` 加 Group + `registry.py` 加一行
> **加引擎** → 新建引擎文件 + 新建 Panel + `main.py` 连线
